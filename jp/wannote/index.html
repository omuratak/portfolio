犬の健康管理
アプリデザイン

UI/UXデザイン

プロジェクトの前提

このプロジェクトは、個人的なプロジェクトとして取り組んだため、リサーチ・デザインなど全てを一人で実施しました。

プロジェクトの概要

背景

私は犬を飼っており、自分の犬の健康管理をもっとしっかり行いたいと考えていました。特に、予防接種や動物病院への通院時のやりとりが全て紙ベースで不便を感じていました。そこで、犬の健康管理をもっとデジタル化できないか？と考えてこのプロジェクトを始めました。

ターゲット

犬を飼っている人（および獣医・動物病院）←今回は犬を飼っているユーザーにフォーカス

犬を飼う側のターゲットの課題

犬の健康に関わるモノが紙媒体なので（ワクチンの接種記録や通院時の処方箋など）、管理が煩雑になりやすい。複数の動物病院に通う場合、カルテが病院間で共有されていないので、事情や通院歴の説明を毎回しないといけないのが不便。通院時に動物の面倒を見ながら、かつ獣医のアドバイスを聞きながら同時にメモを取るのが大変。

獣医側のターゲットの課題（仮説）

（※獣医にはヒアリングできていないので仮説）患者が複数の動物病院に通院する場合、病院間で連携が取れない。飼い主へのヒアリングが飼い主の記憶ベースなので曖昧な情報が多く、診断に苦労することがある。

解決策

犬の健康管理（ワクチン接種記録、通院歴、カルテ、日常の健康記録）をデジタル化し、飼い主と動物病院の連携をよりスムーズにするアプリを設計しました。

プロジェクトのプロセス

人間中心設計のアプローチをベースに、まず「動物病院に行く前後の行動」についてユーザーインタビューを実施し、デザインで解決する課題を定義し、解決策を設計・評価しました。

1. 要求の理解・要件の定義

まず、犬を飼っている2名の知り合いにインタビューを実施し、直近で動物病院に飼い犬を連れて行ったときの体験を詳細にヒアリングしました。さらにその結果をもとにペルソナと現状のジャーニー（AS-IS）を作成しました。

さらに、あるべきジャーニー（TO-BE）を併せて検討することで、あるべき体験の姿をイメージしやすくしました。

これらのインサイトをもとに、以下のように機能要件をまとめました：

  1. 犬の具合についてメモを取る機能
  2. 動物病院での診察を予約する機能
  3. 問診票を事前に共有する機能
  4. 通院後の診察内容・処方箋を見返す機能

2. 解決策の設計・評価

機能要件をもとに、今度はオブジェクト指向で情報設計を行いました。まずオブジェクトを作成し、そのプロパティや取れるアクションを定義し、オブジェクト間の関係性、およびビュー（一覧 vs 詳細画面）を整理しました。

情報設計をベースにして、スケッチを描きました。そのスケッチを使って簡易的なコンセプトテストを実施し、ユーザーフローに大きな問題がないことを確認しました。その後ビジュアルシステムを構築し、スケッチとビジュアルシステムを掛け合わせて、ハイファイのプロトタイプを作成しました。今回は一つの流れに絞ってプロトタイプを作成しました（アカウント登録 → オンボーディング → メモの記入 → 病院の予約 → 予約の変更 → 診察内容の確認 → マイページの確認）。

プロトタイプを使ってユーザビリティテストを実施し、再度ユーザーフローに問題がないことを確認し、またユーザビリティ上の微細な問題を発見してUIを改善しました。概ね分かりやすく自然に使えるUIという評価を得ることができました。

次に向けての学び

今回、オブジェクト指向のUIデザインを心がけ、なるべくモードレスなUIにしようと努めたため、余計な画面が少なく済み、かつユーザーにとって自然と使える分かりやすいデザインにできたと感じました。

実際にこのプロダクトを世に出すとしたら、獣医側の従来の業務フローをデジタル化するにあたっての問題が多く発生すると予想されるため、簡単にはいかないと思いますが、一人の飼い主としての「こんなプロダクトがあったらいいな」という想いを具現化でき、やりがいを感じることができました。
